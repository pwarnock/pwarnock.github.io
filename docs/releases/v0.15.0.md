## üîÑ **Breaking Changes**

### Analytics Requirements

**New Requirement**: All new features must include analytics planning

- **Impact**: Feature development workflow updated
- **Migration**: Existing features should be updated with tracking
- **Timeline**: Immediate for new features, 30 days for existing features

### Pre-Commit Validation

**New Requirement**: Analytics validation runs on every `git push`

- **Impact**: Pushes will fail if interactive elements lack tracking
- **Bypass**: `git push --no-verify` (not recommended)
- **Migration**: Fix tracking attributes or use component integration

### Deployment Process

**New Requirement**: Manual approval gates for infrastructure changes

- **Impact**: Additional validation steps before production deployment
- **Benefit**: Prevents accidental production issues
- **Timeline**: Immediate for all infrastructure changes
